Interview questions

By Kevin Lim

Some misspelled very large number

Question1

Given a list of rectangular buildings, specified as a tuple of 4 x,y corner coordinate pairs (i.e. (x1,y1,x2,y2) for a building that has lower corner x1,y1 and upper corner x2,y2), we want to compute the skyline of the given buildings. Return the skyline as a list of the top edges of the skyline. These will be specified by the starting x and y coordinates, and then the distance to travel across the x axis (i.e. (x, y, d)).

Example. Input [(0,0,2,4), (1,0,3,3)] yields output [(0,4,2), (2,3,1)].

Question2

Given a list of integers, return the smallest positive integer NOT in the list.

Example. Input [5, -3, 0, 3, 2, 1, 8] yields output 4.

Question3

Consider how excel names columns. A, B, C, ..., Z, AA, AB, ..., ZX, ZY, ZZ, AAA, AAB, ... Given an integer, starting from 0, return the excel column name as a string.

Examples. 0 -> A, 25 -> Z, 26 -> AA, 1352 -> AZA

Question 4

An image is specified by a 1d byte array, an integer width(assume divisible by 8) and height. In this format, each pixel is a bit, and can be either 0(white) or 1(black). Given this image, two x coordinates, and a y coordinate, draw a horizontal line between the two x coordinates

Example input would look like:

byte image[64] = {0};
hl(image, 64, 64, 10, 40, 16).

This would draw a horizontal line between (10, 16) to (40, 16) but not change any other part of the image.

Some tome related to faces

Question1

In a revision system, we keep track of revisions by integers. We have a blackbox unit test unitTest(n) that will test and tell us if a revision passes the test or not. Given two integers, one that was a good revision, and one that is a bad revision, find the first revision between them that started failing the unitTest.

Example: Given that the revision history looks as such ... 3(good) 4(good) 5(good) 6(bad) 7(bad) ..., the result when given 3 and 7 should be 6.

Question2

We're given a blackbox function getCategory(n) that categorizes integers to either L, M, or H. Given a list of positive integers, we want to sort this list of integers into the order L first, then M, then H.

Example: getCategory(n) returns L for n<6, M for 6<=n<30, H for n>30. Given [1,5,16,30,4,15], the result should be [1,5,4,16,15,30]
